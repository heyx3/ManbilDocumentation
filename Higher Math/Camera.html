Camera

Represents a basic 2D or 3D camera. Stores its rotation via "forward" and "upwards" vectors.

Member Fields

  • public bool LockUp;
    • If true, keeps the upward vector locked, keeping it immune to pitching.
    • The upward vector will always be affected by rolling and rotation via Quaternion, however.
    • When enabled, the camera's forward vector will be restricted from getting too close to the upward vector via "ClosestDotVariance".
  • public float ClosestDotVariance;
    • If "LockUp" is true, this field indicates the closest the camera can get to the up/down vector in terms of their dot product.
    • Should be between 0 (can look all the way up/down) and 1 (can't pitch the camera at all).
      • The best value is usually close to 0.
  • public ProjectionInfo PerspectiveInfo;
    • The information about this camera's 3D perspective (field of view, clipping planes, and aspect ratio).
    • Only applicable if this is supposed to be a 3D camera.
  • public Vector3f MinOrthoBounds;
    • The information about this camera's 2D perspective (visible rectangular region, relative to the camera).
    • Only applicable if this is supposed to be a 2D camera (or in the rare case you want a 3D camera without the usual 3D perspective).
    • This vector defines the min corner of the visible region relative to the camera.
  • public Vector3f MaxOrthoBounds;
    • The information about this camera's 2D perspective (visible rectangular region, relative to the camera).
    • Only applicable if this is supposed to be a 2D camera (or in the rare case you want a 3D camera without the usual 3D perspective).
    • This vector defines the max corner of the visible region relative to the camera.
  • public Vector3f pos;
    • This camera's position.
  • public Vector3f forward;
    • This camera's forward vector.
  • public Vector3f up;
    • This camera's upwards vector.

Usage

  • This is the recommended base class for any cameras used in-game; it handles all of the necessary math.
